# 题库删除方案

## 目标

题库删除对用户表现为立即消失，同时避免破坏历史答题统计。实现采用软删除，不直接物理清除题库、题目、学习进度和答题记录。

## 权限

- 题库创建者可以删除自己上传的题库。
- 管理员可以删除任意题库，包括旧版本中没有创建者的题库。
- 普通用户不能删除其他用户的题库。
- 管理员可通过后端 `ADMIN_OPENIDS` 配置；本地 mock 联调可设置 `WX_MOCK_ADMIN=true`。

后端负责最终权限校验，前端的 `can_delete` 只用于控制按钮显示，不能替代服务端鉴权。

## 删除行为

调用 `DELETE /api/banks/{bank_id}` 后：

1. 题库状态更新为 `deleted`。
2. 题库内未删除题目统一更新为 `deleted`。
3. 尚未完成的生成任务更新为 `failed`，提示“题库已删除，生成已停止”。
4. 首页、管理页、题库详情、题目和标签接口不再返回该题库的数据。
5. 答题记录和用户进度保留，累计答题、正确率等历史统计不被重置。
6. 原始上传文件暂时保留，便于审计和后续恢复；可另行增加定期物理清理任务。

## 并发处理

后台生成任务会在解析、AI 生成、标签整理和写库前检查题库状态。最终写库使用带 `status != deleted` 条件的更新，避免删除与生成完成并发时把题库重新改回 `ready`。

## 前端交互

- 首页仅在 `can_delete=true` 的题库上显示删除图标。
- 管理页管理员可以删除任意未删除题库。
- 删除前显示题库名称并二次确认。
- 删除期间禁止重复提交；成功后立即从当前列表移除。
